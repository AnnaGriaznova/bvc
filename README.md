# VK Bot v2 - Академии плжяного волейбола BVC

VK чат-бот для платформы дистанционного обучения «Академия BVC». Бот управляет процессом обучения, анкетирования и тестирования студентов пляжного волейбола. Поддерживает **множественное подключение VK-групп** к одному экземпляру бота.

## Функционал

### Для студентов

#### 1. Инструкция
При первом входе в меню пользователь получает инструкцию с:
- Приветствием и ссылкой на учебные материалы (стена группы VK)
- Порядком прохождения курса (видео → тестирование)
- Предупреждением об авторском праве

#### 2. Приветственная анкета (10 вопросов)
- Доступна при `form_first == 1`, кнопка «Приветственная анкета»
- 10 открытых вопросов (ФИО, профессия, спортивный опыт, цели, согласие на публикацию и т.д.)
- Вопросы определены в `form.json`
- Перед стартом — предупреждение о том, что данные не сохранятся при прерывании
- После завершения:
  - `form_first = 2` (завершена)
  - `test_book_1 = 1` (открывается доступ к тестированию)
  - Пользователю отправляется инструкция
  - **Маркетологи (USER_MAR)** получают уведомление с полными ответами

#### 3. Тестирование (Курс 1)
- Доступно при `test_book_1 == 1`, кнопка «Тестирование»
- 20 вопросов, 3 варианта ответа (варианты перемешиваются)
- Проходной балл: 18 из 20
- Вопросы определены в `tests.json`
- При неудаче — кнопка «Пройти заново»
- После успешной сдачи:
  - `test_book_1 = 2` (пройден)
  - `practice_1 = 1` (появляется кнопка «Сдал(а) практику»)
  - **Менеджеры (USER_MEN)** получают уведомление с результатом и кликабельной гиперссылкой на пользователя

#### 4. Сдача практики
- Доступна при `practice_1 == 1`, кнопка «Сдал(а) практику»
- При нажатии:
  - `practice_1 = 2` (кнопка скрывается)
  - **Менеджеры (USER_MEN)** получают уведомление с гиперссылкой и напоминанием открыть доступ к финальной анкете
- Пользователь видит подтверждение: «Ожидайте, вам откроют доступ»

#### 5. Финальная анкета
- Доступна при `access_survey_1 == 1`, кнопка «Финальная анкета»
- Доступ открывается менеджером/админом вручную
- Вопросы определены в `final_form_1.json`
- Поддерживает типы вопросов: `open`, `button`, `rating`, `start`
- После завершения:
  - `access_survey_1 = 2` (доступ закрывается)
  - `diploma_1 = 1` (появляется кнопка «Скачать диплом»)
  - `fortune_wheel + 1` (добавляется вращение)
  - Генерируется и отправляется диплом (PNG)
  - Предлагается «Колесо фортуны»
  - **Маркетологи (USER_MAR)** получают уведомление с полными ответами

##### Структура финальной анкеты:
```
start → 1 → 2 → [2a] → 3 → 4 → [4a] → 5 → 6 → 7 → [7a] → 8 → 9
     → 10 → [10a] → 11 → [11a] → 12 → 13 → [14 → 15] → finish
```

| ID | Тип | Описание | Дальше |
|----|-----|----------|--------|
| start | button | Вступление + кнопка «Начать» | → 1 |
| 1 | open | Время прохождения курса | → 2 |
| 2 | button | Были ли сложности? (Да/Нет) | Да→2a, Нет→3 |
| 2a | open | Описание сложностей | → 3 |
| 3 | open | 3 важных открытия | → 4 |
| 4 | button | Предложения по улучшению (Да/Нет/Не знаю) | Да→4a, Нет/Не знаю→5 |
| 4a | open | Что именно улучшить | → 5 |
| 5 | open | 3 причины выбора BVC | → 6 |
| 6 | open | 3 важные вещи в BVC | → 7 |
| 7 | button | Отметить тренера (Да/Нет) | Да→7a, Нет→8 |
| 7a | open | Кого отметить | → 8 |
| 8 | open | Достижение на курсе | → 9 |
| 9 | open | Цели на 3-6 месяцев | → 10 |
| 10 | rating (1-10) | Оценка курса | 10→11, иначе→10a |
| 10a | open | Чего не хватило до 10 | → 11 |
| 11 | rating (1-10) | Рекомендация друзьям | 10→12, иначе→11a |
| 11a | open | Чего не хватило до 10 | → 12 |
| 12 | button | Публикация в соцсетях (Не против/Против) | → 13 |
| 13 | button | Проверка ФИ и «Кому выдан» (Верно/Изменить ФИ и Кому выдан) | Верно→finish, Изменить→14 |
| 14 | open | Ввод новых ФИ (`user_name`) | → 15 |
| 15 | open | Ввод «Кому выдан» (`komu_vydan`) | → 13 (повторная проверка) |

#### 6. Диплом
- Доступен при `diploma_1 == 1`, кнопка «Скачать диплом»
- Генерируется как PNG на основе шаблона (`sert/sert_X.png`)
- Используется поле `komu_vydan` из БД (по умолчанию «Иванову Ивану»)
- Конфигурация в `diploma_config.json` (позиция, шрифт, цвет)
- Шрифт: `RussoOne-Regular.ttf`

#### 7. Колесо фортуны
- Доступно при `fortune_wheel > 0`, кнопка «Колесо фортуны (N)»
- Кнопка показывается отдельно от кнопок-антагонистов
- После каждого финального анкетирования `+1` вращение
- Менеджер/админ может добавить 1-10 вращений вручную

| Приз | Вероятность | Тип |
|------|-------------|-----|
| 2000 баллов | 1% | Цифровой |
| Вечерняя тренировка | 2% | Цифровой |
| 1000 ₽ на мини-кемп | 4% | Цифровой |
| 3000 ₽ на большой кемп | 4% | Цифровой |
| 300 баллов | 10% | Цифровой |
| 200 баллов | 13% | Цифровой |
| 100 баллов | 14% | Цифровой |
| Чек-лист по элементам | 15% | Цифровой |
| Ручка | 15% | Физический |
| Браслет | 5% | Физический |
| Брелок | 5% | Физический |

- **Физические призы** (Браслет, Ручка, Брелок) — пользователь получает инструкцию подойти к тренеру
- **Цифровые призы** — пользователь получает текст «менеджер свяжется», **менеджеры (USER_MEN)** получают уведомление с гиперссылкой и названием приза

### Панели управления

#### Админ-панель (USER_ADMIN)
Кнопка «АДМИН» в меню. Функции:
- **Скачать базу** — экспорт всех пользователей в XLSX
- **Загрузить базу** — импорт пользователей из XLSX (создание/обновление)
- **Открыть доступ к финальной анкете** — поиск → выбор курса → открыть/закрыть
- **Добавить вращений колеса фортуны** — поиск → выбор количества (1-10)
- **Посмотреть ответы на анкеты** — поиск → выбор анкеты → просмотр ответов

#### Панель менеджера (USER_MEN)
Кнопка «Менеджер» в меню. Функции:
- **Открыть доступ к финальной анкете** — поиск → выбор курса → открыть/закрыть
- **Изменить поле Кому выдан** — поиск → отображение текущего значения → ввод нового
- **Добавить вращений колеса фортуны** — поиск → выбор количества (1-10)

#### Панель маркетинга (USER_MAR)
Кнопка «Маркетинг» в меню. Функции:
- **Посмотреть ответы на анкеты** — поиск → выбор анкеты → просмотр ответов

#### Общая логика управления доступом
1. Поиск пользователя по части имени (регистронезависимый)
2. Пагинация по 6 пользователей на страницу
3. Выбор курса (1-4)
4. Проверка: практика должна быть сдана (`practice_X = 2`)
5. Открытие/закрытие доступа → автоматическое уведомление пользователя

## Мультигрупповая поддержка

Бот поддерживает подключение нескольких VK-групп к одному экземпляру. Каждая группа имеет свой токен и confirmation token.

### Переменные окружения для групп
```
GROUP_1=236786685
TOKEN_1=vk1.a...
CONFIRMATION_TOKEN_1=fb5c3aa4

GROUP_2=237156222
TOKEN_2=vk1.a...
CONFIRMATION_TOKEN_2=75f8f141
```

Индексация начинается с 1. Группы добавляются последовательно (GROUP_3, TOKEN_3 и т.д.).

### Логика маршрутизации
1. При старте — парсятся все `GROUP_X`/`TOKEN_X`/`CONFIRMATION_TOKEN_X`
2. Для каждой группы создаётся отдельный экземпляр `VKAPI`
3. При входящем запросе извлекается `group_id` из JSON:
   - **confirmation** → возвращается `CONFIRMATION_TOKEN_X` соответствующей группы
   - **message_new** → переключается на `VKAPI` этой группы, все хендлеры работают через него
   - Неизвестный `group_id` → WARNING в лог, возврат `"ok"`

### Backward compatibility
Если `GROUP_1` не задан, бот пытается использовать старые переменные `TOKEN` и `CONFIRMATION_TOKEN`.

### Логика переходов между состояниями

| Событие | Изменения в БД | Уведомление |
|---------|---------------|-------------|
| Новый пользователь | `form_first = 1` | — |
| Приветственная анкета заполнена | `form_first = 2`, `test_book_1 = 1` | USER_MAR (ответы) |
| Тест сдан (≥18/20) | `test_book_1 = 2`, `practice_1 = 1` | USER_MEN (результат) |
| Тест не сдан (<18/20) | — | USER_MEN (результат) |
| Нажата «Сдал(а) практику» | `practice_1 = 2` | USER_MEN (напоминание) |
| Менеджер открыл доступ | `access_survey_X = 1` | Пользователь |
| Финальная анкета завершена | `access_survey_X = 2`, `diploma_X = 1`, `fortune_wheel + 1` | USER_MAR (ответы) |
| Цифровой приз (не физический) | `fortune_wheel - 1` | USER_MEN (приз) |
| Физический приз | `fortune_wheel - 1` | — (тренеру на месте) |

### Динамическое меню — кнопки-антагонисты

В меню показывается только одна кнопка-антагонист (приоритет сверху вниз):

| Приоритет | Условие | Кнопка |
|-----------|---------|--------|
| 1 | `form_first == 1` | «Приветственная анкета» |
| 2 | `test_book_1 == 1` | «Тестирование» |
| 3 | `practice_1 == 1` | «Сдал(а) практику» |
| 4 | `access_survey_1 == 1` | «Финальная анкета» |
| 5 | `diploma_1 == 1` | «Скачать диплом» |

Дополнительно отображаются:
- **«Колесо фортуны (N)»** — если `fortune_wheel > 0` (отдельная строка)
- **«Менеджер»** — если `user_id in USER_MEN_IDS`
- **«Маркетинг»** — если `user_id in USER_MAR_IDS`
- **«АДМИН»** — если `user_id == USER_ADMIN_ID`

## In-memory сессии

Все активные сессии хранятся в словарях в оперативной памяти (сбрасываются при перезапуске):

| Сессия | Ключ | Описание |
|--------|------|----------|
| `USER_SESSIONS` | `user_id` | Текущий тест (variant, question, score, shuffled_answers) |
| `FORM_SESSIONS` | `user_id` | Приветственная анкета (question, answers) |
| `FINAL_FORM_SESSIONS` | `user_id` | Финальная анкета (course, current_question, answers) |
| `ADMIN_SEARCH_SESSIONS` | `admin_id` | Поиск в админке (step, results, page, selected_user, mode) |
| `FORTUNE_WHEEL_SESSIONS` | `user_id` | Колесо фортуны (course, prize) |

## Правила написания кода

### ⚠️ Тексты сообщений

**Все текстовые сообщения хранятся в `texts.json`.**

```python
# ❌ Неправильно:
message = f"Пользователь {user_name} сдал тест!"

# ✅ Правильно:
msg = TEXTS_DATA.get("key", "default text")
message = msg.format(user_name=user_name)
```

### Архитектура
- **Webhook**: VK Callback API (не long polling)
- **Async**: asyncio + aiohttp + asyncpg
- **Multi-group**: один сервер — несколько VK-групп

### Стилистика

1. **Именование**: функции — `snake_case`, классы — `PascalCase`, константы — `UPPER_SNAKE_CASE`
2. **Типизация**: type hints для всех функций
3. **Обработка ошибок**: try-except на все внешние вызовы, `flush=True` в print
4. **Клавиатуры**: отдельные функции-билдеры, `one_time: False`, `inline: False`
5. **Сессии**: словари `*_SESSIONS`, очистка при выходе в меню

### Обработчики сообщений

```python
# 1. Проверка ролей
is_admin = (user_id == USER_ADMIN_ID)
is_manager = (user_id in USER_MEN_IDS)
is_marketing = (user_id in USER_MAR_IDS)

# 2. Обработка кнопок (по приоритету)
if text.lower() == "кнопка":
    return

# 3. Обработка текста в сессии
if user_id in SOME_SESSION:
    await handle_session(user_id, peer_id, text)
    return

# 4. Игнорирование неизвестных сообщений
```

## Развертывание

### Amvera Cloud
1. Загрузить код в репозиторий
2. Настроить переменные окружения (`GROUP_X`, `TOKEN_X`, `CONFIRMATION_TOKEN_X`, `USER_*`, `DB_*`)
3. Автоматическое развертывание на порту 8080

### VK настройка (для каждой группы)
1. Создать/настроить группу VK
2. Включить Callback API в разделе «Управление → API»
3. Указать URL: `https://your-domain.amvera.io`
4. Скопировать Confirmation Token → в `CONFIRMATION_TOKEN_X`
5. Создать токен группы с правами: messages, docs, photos → в `TOKEN_X`
6. Добавить `GROUP_X` с ID группы
